# 성장형 놀리지에셋 & 메모리 프로토콜 (쓸수록 똑똑해지는 규칙)

목표: 대화·분석을 거듭할수록 에이전트가 **계정/클라이언트 맥락과 교훈을 자동 축적**해
더 빠르고 정확해진다. 히스토리를 알아서 파악·기록한다.

## 자산 구조
```
marketing/
  METRICS.md            # 지표·정합성 규칙 (공용)
  CREATIVE_PLAYBOOK.md  # 소재 기획/생성/점검 (공용)
  knowledge/
    _GLOBAL.md          # 범용 교훈(모든 계정 공통)
    <account>.md        # 계정별: 지표 기준선, 통하는 앵글/후크, 금칙어, 리포트 포맷, 특이사항
  history/
    <account>-log.md    # 대화/분석 이력 요약(날짜순), 의사결정·결과
```

## 언제·무엇을 기록하나 (자동)
매 분석/대화 종료 시 다음을 **짧게** 남긴다(반성 사이클):
1. **사실/기준선**: 새로 알게 된 계정 지표 기준선(평균 CTR/CPA/ROAS), 타깃, 예산, 시즌성 → `knowledge/<account>.md`.
2. **통한 것/실패한 것**: 어떤 앵글·후크·포맷·세그먼트가 성과↑/↓ → 계정 파일 + 범용이면 `_GLOBAL.md`.
3. **정합성 이슈**: 발견한 데이터 불일치·플랫폼 특이(타임존/윈도우/통화) → 다음에 먼저 점검하도록 기록.
4. **의사결정 로그**: 무엇을 왜 추천/실행했고 결과는 어땠나 → `history/<account>-log.md`.
5. **리포트 포맷 선호**: 유저가 원하는 표/순서/지표 → 계정 파일(다음엔 그 형식으로 자동).

## 기록 원칙
- **검증된 것만 승격**: 추측이 아니라 *정합성 검산을 통과한* 수치·교훈만 자산에 올린다.
- **한 파일=한 맥락**, 한 줄 요약 + 근거. 중복은 갱신(새 파일 X).
- **상대 날짜는 절대 날짜로** 변환 기록.
- 매 세션 시작 시 해당 계정의 `knowledge/<account>.md` + 최근 `history`를 먼저 읽고 맥락 복원.

## 복리 효과
세션 N: 기준선·통하는 앵글·금칙어·선호 포맷이 이미 자산에 → 기획/분석/점검을 **즉시·정확**하게.
→ 이것이 Hermes "grows with you"를 **증거기반(검산 통과분만 누적)** 으로 구현한 형태.
